**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic services](1.1-week1/) - creating an AWS Free Tier account, setting up security, and learning S3, RDS, Lambda, EC2, and Bedrock.

**Week 2:** [Exploring Kiro IDE/CLI and AWS foundation concepts](1.2-week2/) - practicing VPC, CloudFormation, EC2, and the AWS Well-Architected Framework.

**Week 3:** [Practicing basic AWS networking and security](1.3-week3/) - learning Route 53, VPC, RDGW, VPN, and network configurations.

**Week 4:** [Researching and practicing storage and backup services](1.4-week4/) - learning Transit Gateway, EC2, AWS Backup, Storage Gateway, and S3/CloudFront.

**Week 5:** [Practicing storage, import/export, and FSx](1.5-week5/) - working with S3, Snow Family, Storage Gateway, EC2 import/export, and Amazon FSx.

**Week 6:** [Exploring AWS security and automation](1.6-week6/) - studying FSx, IAM, Cognito, AWS Organization, Security Hub, and Lambda for EC2.

**Week 7:** [Building infrastructure and managing access on AWS](1.7-week7/) - deploying VPC, EC2, RDS, Load Balancer, Auto Scaling, and IAM.

**Week 8:** [Reporting project progress and practicing AWS security](1.8-week8/) - working with IAM, KMS, S3, CloudTrail, Athena, and participating in a community event.

**Week 9:** [Reviewing databases and practicing data migration](1.9-week9/) - studying RDS, Aurora, Redshift, and migration tools.

**Week 10:** [Implementing ETL, data analysis, and visualization](1.10-week10/) - practicing Glue, Athena, QuickSight, and testing the project.

**Week 11:** [Team meeting to report project progress](1.11-week11/).

**Week 12:** [Completing and writing the project report](1.12-week12/).
