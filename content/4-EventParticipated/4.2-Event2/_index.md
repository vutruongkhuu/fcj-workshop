### Event Purpose

- **Shift AI thinking from surface-level to practical:** the event aimed to help attendees understand that AI becomes truly useful only when backed by concrete context and the right architecture (such as Multi-Agent models or understanding LLM uncertainty), instead of just writing basic prompts.
- **Democratize mastery of data and cloud infrastructure:** introduce modern technology solutions that optimize operations—from using No-code/Low-code tools (Amazon Quick) for everyone to analyze data, to optimizing cost, performance, and security at scale with Amazon CloudFront.
- **Guide young IT talent to stay grounded:** the event both inspired innovation (through the 36-hour Hackathon experience of team UTMorpho) and reminded the market reality that, despite powerful AI, fundamental skills like Backend Engineering, Terraform (Infrastructure as Code), and building safe, reliable systems remain core values employers seek.

### What I Learned

- **“Context is King” in AI:** AI only truly solves problems when you provide real project or business context. You need to build a “Second Brain” and supply it with project-specific data and context for AI to return accurate results.
- **Understand LLM behavior to debug effectively:** It’s important to realize that Temperature = 0 does not ensure identical output every time because of hardware inference optimizations. This insight helps avoid surprises and build strategies to control errors in AI applications.
- **No-code and Multi-Agent trends for data work:** I learned how natural language can drive data tasks (via Amazon Quick) and how advanced system design can use multiple AI agents working together to solve complex problems (such as credit scoring).
- **Web infrastructure survival skills:** I understood how to use Amazon CloudFront to deliver content at the edge, optimize costs, speed up page loads, and secure systems. This knowledge is especially useful for Web Developers or DevOps/Platform Engineers.
- **Real-world hackathon lessons:** I learned how to manage time and collaborate so a product can go from a raw idea to a working demo in 36 hours, especially the courage to pivot when facing technical blocks.
- **AI is important, but fundamentals remain essential:** An eye-opening lesson from the experts was that no matter how advanced AI becomes, companies still prioritize candidates strong in traditional Backend Engineering, system thinking, and Terraform. AI is a bonus; foundational technical ability to build safe, reliable systems is what matters most.

### How I Will Apply It

#### 1. Upgrade AI work practices (Context & Agent mindset)

- **Build a “Second Brain” for projects:** Instead of typing prompts from scratch each time, prepare a Markdown file with full project context (goals, tech stack, database structure, naming conventions). Before asking AI to fix bugs or write new code, feed that context first.
- **Automate reporting with no-code:** If your work involves reporting or monitoring metrics (such as website analytics or system logs), explore AI-powered BI tools like Amazon Quick or equivalents to extract dashboards from natural language instead of writing SQL or Excel formulas manually.

#### 2. Optimize your web project (CloudFront mindset)

- **Integrate CDN for products:** If you build a website (e.g. streaming site, e-commerce site), consider fronting it with Amazon CloudFront or Cloudflare.

#### 3. Strengthen backend and infrastructure fundamentals (Expert advice)

- **Learn and use Terraform instead of manual clicks:** When deploying apps to the cloud (AWS/Azure), stop creating servers or databases via web consoles and practice writing Terraform instead.
- **Handle LLM uncertainty safely:** If your app calls OpenAI/Gemini APIs, do not assume the result will always follow the expected format, even with Temperature = 0.

#### 4. Teamwork and deadline skills (Hackathon lesson)

- **Apply a pivot mindset:** When working in a team and encountering a hard bug or feature that keeps failing near a deadline, be willing to pivot like UTMorpho.

### Event Experience

Attending the **“Conference Call”** workshop was a very valuable experience, giving me a comprehensive view of modernizing applications and databases with modern methods and tools. Highlights included:

#### 🎭 1. Learning from speakers
- **Practical mindset:** Gained deep perspectives from top experts (AWS, GoTymeX, Cloud Kinetics...) on 2026 AI and Cloud trends.
- **Real market insight:** Recognized the technology paradox that as AI makes software cheaper, demand for engineers who operate systems well grows.
- **Stay grounded:** A hard truth from experts: “No matter how strong AI is, the core of an engineer is still traditional Backend skills and system thinking.”

#### 💻 2. Technical experience
- **LLM fundamentals:** Learned about the token-selection mechanism of large language models and the uncertainty that remains even at Temperature = 0 due to hardware inference optimization.
- **Multi-Agent architecture:** Saw an enterprise-style multi-agent design for a virtual credit committee, solving complex problems with cooperating AI agents that cross-check each other.
- **Infrastructure-as-code (IaC):** Understood why using Terraform is essential to deploy, track, and replicate infrastructure instead of manual “point-and-click” setup.

#### 🛠️ 3. Modern tool adoption
- **Amazon CloudFront:** Experienced how to optimize website performance, deliver content via edge locations, reduce origin server load, enhance security, and save bandwidth cost.
- **Amazon Quick ecosystem:** Explored no-code/low-code trends with tools that let you query data in natural language.

#### 🤝 4. Networking and collaboration
- **Community space:** Had opportunities to connect on floors 26 and 36 with hundreds of students, developers, and systems engineers at the FCAJ Community Day.
- **Peer inspiration:** Learned directly from team UTMorpho (LotusHacks 2026) about how they survived, managed time, and overcame the pressure of a 36-hour development cycle.

#### 📝 5. Key takeaways
- **Context is King:** To make AI truly useful, provide enough project/business context instead of vague prompts. Always build a backend validation layer for AI outputs.
- **Build standard systems:** A good system is not just “working”; it must ensure three qualities: secure, reliable, and cost efficient.
- **Be brave to pivot:** Under deadlines or in hackathons, if you encounter a technical dead-end, decide to trim features or change direction to deliver a working MVP for demo.
